# Security Policy

## Supported Versions

This repository is maintained for academic and reproducibility purposes. Security fixes are provided on a best-effort basis for the default branch.

## Reporting a Vulnerability

If you discover a security issue, please do not disclose it publicly first.

1. Open a private communication channel with the maintainers when possible.
2. If private contact is not available, open a minimal public issue without exploit details and request maintainer follow-up.
3. Include reproduction steps, potential impact, and affected files.

## Response Expectations

- Initial acknowledgement target: 3-5 business days.
- Triage and mitigation timeline depends on severity and maintainers' availability.

## Dependency Hygiene

Dependencies are pinned with compatible ranges in `requirements.txt`. Please report vulnerable packages with a CVE reference when possible.
